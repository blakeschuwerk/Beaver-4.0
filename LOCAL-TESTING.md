# Local Pipeline Testing

Run the full Beaver pipeline on your Mac — real scraping, Docling extraction, and local Qwen 2.5 7B classification — with **zero GCP cost**.

## Prerequisites

- Node 20+, pnpm 9+, Python 3.11+
- ~12GB+ RAM (16GB recommended for `qwen2.5:7b`)
- First run downloads large deps (Ollama model, Docling, Playwright Chromium)

## One-time setup

```bash
pnpm install && pnpm build

# Python virtualenv + heavy deps (scraper + analyzer)
python3 -m venv .venv
.venv/bin/pip install -r functions/scraper/requirements.txt \
                   -r functions/scraper/requirements-scraping.txt \
                   -r functions/analyzer/requirements.txt \
                   -r functions/analyzer/requirements-extraction.txt
.venv/bin/python -m playwright install chromium

# Local Qwen 2.5 7B via Ollama (writes .env.local)
pnpm qwen:setup
```

## Run the full local pipeline

```bash
# Live scrape of 2 counties + Docling + local Qwen (needs network, ~5–15 min first run)
pnpm local:run

# Quick smoke test without county network access (local PDF + Qwen)
pnpm local:run:demo
```

This runs two stages:

1. **`local:scrape-extract`** — scrapes 2 counties from [config/counties.json](config/counties.json), downloads PDFs, extracts with Docling
2. **`local:classify`** — classifies chunks with local Qwen 2.5 7B, matches against [config/test-user.json](config/test-user.json)

For Playwright (crawl4ai), browsers install to `.playwright-browsers/` in the repo:

```bash
PLAYWRIGHT_BROWSERS_PATH=$PWD/.playwright-browsers .venv/bin/python -m playwright install chromium
```

## Outputs

All written under `local-run/` (**tracked in git** so you can clone and continue on another machine):

| Path | Contents |
|------|----------|
| `local-run/raw/<county>/<doc>/` | Downloaded PDFs |
| `local-run/staging/<county>/<doc>/chunks.json` | Extracted chunks |
| `local-run/projects.jsonl` | Classified projects |
| `local-run/matches.jsonl` | User matches |
| `local-run/scrape-summary.json` | Per-county scrape results |
| `local-run/classify-summary.json` | Classification summary |

## Configuration

`.env.local` (auto-generated by `pnpm qwen:setup`):

| Variable | Purpose |
|----------|---------|
| `LLM_ENDPOINT_URL` | Ollama OpenAI-compatible endpoint |
| `LLM_MODEL` | `qwen2.5:7b` or `qwen2.5:3b` (auto-selected by RAM) |
| `SCRAPER_REAL` | `true` — use civic-scraper/crawl4ai |
| `USE_DOCLING` | `true` — real PDF extraction |
| `LOCAL_MAX_DOCS_PER_COUNTY` | Cap downloads per county (default 3) |

## Swap models

```bash
ollama pull qwen2.5:3b    # faster, less RAM
# Edit LLM_MODEL in .env.local
```

## Moving to the cloud

When local quality looks good, follow [OUTLETS.md](OUTLETS.md) Bundle A→C to deploy to `beaver4` with a hosted LLM (or tunnel this same Ollama instance).

## Troubleshooting

- **No PDFs downloaded** — run `pnpm check:county-links`; county site may have changed
- **Docling OOM** — set `USE_DOCLING=false` in `.env.local` temporarily (uses mock text)
- **Ollama slow** — use `qwen2.5:3b`; reduce `LOCAL_MAX_DOCS_PER_COUNTY=1`
- **civic-scraper empty** — check `platform` field matches site vendor
